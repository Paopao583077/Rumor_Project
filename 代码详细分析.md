## 阶段 0：数据预处理 (preprocess.py)

### 功能概述
将杂乱的原始 TXT 文件转换为整齐的 CSV 表格，供后续训练使用。

### 原始数据格式

**TXT 文件结构**（每 3 行为一条数据）：
```

http://example.com/image1.jpg|null                ← 第 2 行：图片 URL
今天天气真好，大家出门要注意防晒哦！              ← 第 3 行：微博正文
```

#### 2. 内部处理函数 `process_file`

**Step 1：读取文件**
```python

    lines = f.readlines()
```

**Step 2：按 3 行一组循环处理**
```python
for i in range(0, len(lines), 3):  # 步长为 3
    if i + 2 >= len(lines): break  # 防止越界
```

**Step 3：提取图片路径**
```python

image_urls = image_line.split('|')

# 选择第一张有效图片

for url in image_urls:
    if url != 'null' and url.endswith('.jpg'):
        target_image_name = url.split('/')[-1]  # 提取文件名
        break

# 过滤无图数据（多模态必须有图）
```

## 阶段 2：模型定义 (model.py)

### 功能概述
定义双塔架构的谣言检测模型，包括 BERT 文本编码器、ResNet 图像编码器和文本引导的注意力融合模块。

### 核心组件 1：TextGuidedAttention（文本引导的视觉注意力）

这是项目的**核心创新点**。

#### 初始化

```python
class TextGuidedAttention(nn.Module):
    def __init__(self, text_dim, img_dim, hidden_dim):
        super(TextGuidedAttention, self).__init__()
        self.text_proj = nn.Linear(text_dim, hidden_dim)    # 文本投影：768 → 256
        self.img_proj = nn.Linear(img_dim, hidden_dim)      # 图像投影（Key）：2048 → 256
        self.v_proj = nn.Linear(img_dim, hidden_dim)        # 图像投影（Value）：2048 → 256
        self.attention_score = nn.Linear(hidden_dim, 1)     # 注意力分数计算
        self.tanh = nn.Tanh()
        self.softmax = nn.Softmax(dim=1)
```

#### 前向传播

```python
def forward(self, text_feat, img_feat):
    # text_feat: (batch, 768)      ← BERT 输出
    # img_feat:  (batch, 49, 2048) ← ResNet 的 7×7 区域特征
    
    # Step 1：投影到统一空间
    t_proj = self.text_proj(text_feat).unsqueeze(1)  # (batch, 1, 256)
    i_proj = self.img_proj(img_feat)                 # (batch, 49, 256)
    
    # Step 2：计算注意力分数
    combined = self.tanh(t_proj + i_proj)            # (batch, 49, 256)
    scores = self.attention_score(combined).squeeze(-1)  # (batch, 49)
    alpha = self.softmax(scores)                     # (batch, 49) 归一化为权重
    
    # Step 3：加权求和
    v = self.v_proj(img_feat)                        # (batch, 49, 256)
    context_img = (alpha.unsqueeze(-1) * v).sum(dim=1)  # (batch, 256)
    
    return context_img  # 文本引导的增强视觉特征
```

**数学原理**：
- `alpha[i]` 表示第 i 个图像区域的重要性（0-1 之间，总和为 1）
- 如果文本提到"火灾"，模型会自动关注图片中"火焰"的区域

### 核心组件 2：RumorDetector（完整模型）

#### 初始化

```python
class RumorDetector(nn.Module):
    def __init__(self, num_classes=2):
        super(RumorDetector, self).__init__()
        
        # 1. BERT 文本编码器
        self.bert = BertModel.from_pretrained('bert-base-chinese')
        
        # 2. ResNet 图像编码器
        resnet = models.resnet50(pretrained=True)
        modules = list(resnet.children())[:-2]  # 去掉池化层和全连接层
        self.resnet_features = nn.Sequential(*modules)
        
        # 3. 注意力融合模块
        self.fusion_attention = TextGuidedAttention(768, 2048, 256)
        
        # 4. 分类器
        self.classifier = nn.Sequential(
            nn.Linear(768 + 256, 128),  # 拼接后的特征 → 128 维
            nn.ReLU(),
            nn.Linear(128, 2)           # 2 分类（真实/谣言）
        )
```

#### 前向传播

```python
def forward(self, input_ids, attention_mask, image):
    # Step 1：提取文本特征
    bert_out = self.bert(input_ids=input_ids, attention_mask=attention_mask)
    text_feat = bert_out.pooler_output  # (batch, 768)
    
    # Step 2：提取图像特征
    img_feat_map = self.resnet_features(image)  # (batch, 2048, 7, 7)
    batch, c, h, w = img_feat_map.size()
    img_feat_seq = img_feat_map.view(batch, c, -1).permute(0, 2, 1)
    # (batch, 2048, 49) → (batch, 49, 2048)
    
    # Step 3：注意力融合
    attended_img = self.fusion_attention(text_feat, img_feat_seq)
    # (batch, 256) ← 文本引导的视觉特征
    
    # Step 4：分类
    final_feat = torch.cat((text_feat, attended_img), dim=1)  # (batch, 1024)
    logits = self.classifier(final_feat)  # (batch, 2)
    
    return logits
```

**输出示例**：
```python
logits = [[-2.3, 3.1], [1.5, -0.8], ...]  # 未归一化的分数
# 经过 Softmax 后变为概率：[[0.05, 0.95], [0.92, 0.08], ...]
```

## 阶段 3：模型训练 (train.py)

### 功能概述
使用训练集数据训练模型，通过反向传播更新参数。

### 核心流程

#### 1. 环境检测

```python
DEVICE = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print(f"🚀 正在使用计算设备: {DEVICE}")
```

#### 2. 超参数设置

```python
BATCH_SIZE = 8   # 每次喂给模型 8 个样本
LR = 2e-5        # 学习率（BERT 微调的标准值）
EPOCHS = 3       # 训练 3 轮
```

**为什么学习率这么小？**
- BERT 已经预训练过，参数已经很好
- 用大学习率会"破坏"原有知识
- 2e-5 是论文推荐的微调学习率

#### 3. 数据加载

```python
train_ds = WeiboDataset('./data/train.csv', mode='train')
val_ds = WeiboDataset('./data/test.csv', mode='val')

train_loader = DataLoader(train_ds, batch_size=BATCH_SIZE, shuffle=True)
val_loader = DataLoader(val_ds, batch_size=BATCH_SIZE, shuffle=False)
```

**DataLoader 的作用**：
- 自动分批（batch）
- 自动打乱（shuffle）
- 多线程加载（加速）

#### 4. 模型初始化

```python
model = RumorDetector().to(DEVICE)
criterion = nn.CrossEntropyLoss()  # 交叉熵损失
optimizer = torch.optim.AdamW(model.parameters(), lr=LR)  # AdamW 优化器
```

#### 5. 训练循环

```python
for epoch in range(EPOCHS):
    model.train()  # 启用训练模式
    total_loss = 0
    
    for i, batch in enumerate(train_loader):
        # 数据搬到 GPU
        input_ids = batch['input_ids'].to(DEVICE)
        mask = batch['attention_mask'].to(DEVICE)
        images = batch['image'].to(DEVICE)
        labels = batch['label'].to(DEVICE)
        
        # 前向传播
        optimizer.zero_grad()
        logits = model(input_ids, mask, images)
        loss = criterion(logits, labels)
        
        # 反向传播
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item()
        
        if i % 10 == 0:
            print(f"Epoch {epoch + 1} | Step {i}/{len(train_loader)} | Loss: {loss.item():.4f}")
    
    print(f"=== Epoch {epoch + 1} 完成 | 平均 Loss: {total_loss / len(train_loader):.4f} ===")
```

**损失计算示例**：
```python
# 假设 batch_size=2
logits = [[2.1, -0.5],   # 模型预测第 1 个样本是真实的
          [-1.3, 3.2]]   # 模型预测第 2 个样本是谣言
labels = [1, 1]          # 真实标签：都是谣言

# CrossEntropyLoss 会计算：
# 第 1 个样本预测错了 → loss 很大
# 第 2 个样本预测对了 → loss 很小
```

#### 6. 保存模型

```python
torch.save(model.state_dict(), 'best_model.pth')
print("💾 训练完成，模型已保存为 best_model.pth")
```

```python
        # 更新预览
```

#### 4. 预测功能

```python
def predict(self):
    text = self.text_input.get("1.0", tk.END).strip()
    
    # 处理图片
    image = Image.open(self.image_path).convert('RGB')
    image_tensor = transform(image).unsqueeze(0).to(DEVICE)
    
    # 处理文本
    encoding = tokenizer.encode_plus(
        text, add_special_tokens=True, max_length=128,
        padding='max_length', truncation=True, return_tensors='pt'
    )
    input_ids = encoding['input_ids'].to(DEVICE)
    attention_mask = encoding['attention_mask'].to(DEVICE)
    
    # 预测
    with torch.no_grad():
        logits = model(input_ids, attention_mask, image_tensor)
        probs = torch.softmax(logits, dim=1).squeeze(0)
        
        prob_non_rumor = probs[0].item()
        prob_rumor = probs[1].item()
        
        pred = 1 if prob_rumor > prob_non_rumor else 0
    
    # 更新界面
    self._update_results(pred, prob_non_rumor, prob_rumor)
```

#### 5. 结果显示

```python
def _update_results(self, pred, prob_non_rumor, prob_rumor):
    if pred == 1:
        result_str = "🔴 判决结果：谣言 (RUMOR)"
        color = "#ff0000"
    else:
        result_str = "🟢 判决结果：非谣言 (NON-RUMOR)"
        color = "#008000"
    
    self.result_text.set(result_str)
    self.result_label.config(foreground=color)
    
    # 更新进度条
    self.prob_rumor_bar['value'] = prob_rumor * 100
    self.prob_nonrumor_bar['value'] = prob_non_rumor * 100
```

## 数学原理深度解析

### 1. BERT 文本编码数学原理

#### 1.1 Token Embedding（词嵌入）

输入文本首先被分词，然后转换为向量：

$$\mathbf{E}_{token} = \text{Embedding}(\text{token\_ids})$$

其中每个 token 被映射到一个 768 维向量。

#### 1.2 Multi-Head Self-Attention（多头自注意力）

**核心公式**：
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**具体计算步骤**：

1. **线性投影**（生成 Query, Key, Value）：
   $$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$

2. **计算相似度矩阵**（点积）：
   $$S = \frac{QK^T}{\sqrt{d_k}}$$
   - $d_k = 64$（每个头的维度）
   - 除以 $\sqrt{d_k}$ 是为了稳定梯度

3. **归一化为注意力权重**：
   $$\alpha = \text{softmax}(S)$$
   - 每一行的权重和为 1

4. **加权求和得到输出**：
   $$\text{Output} = \alpha V$$

#### 1.3 [CLS] Token 的语义表示

BERT 输出中，第一个位置的 [CLS] token 聚合了整句话的语义：

$$\mathbf{h}_{text} = \text{BERT}(\text{input\_ids})_{[CLS]} \in \mathbb{R}^{768}$$

---

### 2. ResNet 图像编码数学原理

#### 2.1 卷积操作

对于输入图像 $I \in \mathbb{R}^{H \times W \times C}$，卷积操作为：

$$\mathbf{F}_{out}(i,j) = \sum_{m}\sum_{n}\sum_{c} I(i+m, j+n, c) \cdot K(m, n, c)$$

其中 $K$ 是卷积核。

#### 2.2 残差连接（Residual Connection）

ResNet 的核心创新是跳跃连接：

$$\mathbf{x}_{l+1} = \mathcal{F}(\mathbf{x}_l, \{W_l\}) + \mathbf{x}_l$$

这使得网络可以学习"残差"而不是直接映射，解决了梯度消失问题。

#### 2.3 特征图提取

ResNet-50 在移除全局池化层后，输出特征图：

$$\mathbf{F}_{img} = \text{ResNet}_{conv}(I) \in \mathbb{R}^{2048 \times 7 \times 7}$$

将其重塑为序列形式：

$$\mathbf{F}_{seq} = \text{reshape}(\mathbf{F}_{img}) \in \mathbb{R}^{49 \times 2048}$$

每个位置代表图像的一个区域特征。

---

### 3. 文本引导的注意力机制数学原理

这是本项目的**核心创新**，采用类似 Bahdanau Attention 的机制。

#### 3.1 维度对齐（线性投影）

将不同维度的特征投影到统一空间：

$$Q = \mathbf{h}_{text}W_Q + b_Q \in \mathbb{R}^{256}$$
$$K = \mathbf{F}_{seq}W_K + b_K \in \mathbb{R}^{49 \times 256}$$
$$V = \mathbf{F}_{seq}W_V + b_V \in \mathbb{R}^{49 \times 256}$$

其中：
- $W_Q \in \mathbb{R}^{768 \times 256}$
- $W_K, W_V \in \mathbb{R}^{2048 \times 256}$

#### 3.2 计算注意力分数

**步骤 1：相加并激活**

$$\mathbf{e}_i = \tanh(Q + K_i), \quad i=1,2,...,49$$

这里使用广播机制，$Q$ 被扩展为 $(49, 256)$。

**步骤 2：计算标量分数**

$$s_i = w^T\mathbf{e}_i$$

其中 $w \in \mathbb{R}^{256}$ 是可学习参数。

**步骤 3：归一化为权重**

$$\alpha_i = \frac{\exp(s_i)}{\sum_{j=1}^{49}\exp(s_j)}$$

约束条件：$\sum_{i=1}^{49}\alpha_i = 1$，且 $\alpha_i \in [0,1]$

#### 3.3 加权求和得到上下文向量

$$\mathbf{v}_{att} = \sum_{i=1}^{49} \alpha_i \cdot V_i \in \mathbb{R}^{256}$$

**物理意义**：
- $\alpha_i$ 高 → 第 $i$ 个图像区域与文本高度相关
- 例如文本提到"火灾"，则包含"火焰"的区域权重会变大

---

### 4. 特征融合与分类数学原理

#### 4.1 特征拼接

$$\mathbf{v}_{final} = [\mathbf{h}_{text}; \mathbf{v}_{att}] \in \mathbb{R}^{1024}$$

其中 $[;]$ 表示拼接操作：$768 + 256 = 1024$

#### 4.2 多层感知机分类器

$$\mathbf{h}_1 = \text{ReLU}(W_1\mathbf{v}_{final} + b_1) \in \mathbb{R}^{128}$$
$$\mathbf{z} = W_2\mathbf{h}_1 + b_2 \in \mathbb{R}^{2}$$

其中：
- $W_1 \in \mathbb{R}^{128 \times 1024}$
- $W_2 \in \mathbb{R}^{2 \times 128}$

#### 4.3 Softmax 归一化

$$P(y=k | x) = \frac{\exp(z_k)}{\sum_{j=0}^{1}\exp(z_j)}$$

**输出示例**：
$$\mathbf{z} = [2.3, -1.5]$$
$$P(\text{非谣言}) = \frac{\exp(2.3)}{\exp(2.3)+\exp(-1.5)} = 0.95$$
$$P(\text{谣言}) = \frac{\exp(-1.5)}{\exp(2.3)+\exp(-1.5)} = 0.05$$

---

### 5. 训练优化数学原理

#### 5.1 交叉熵损失函数

对于二分类问题：

$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N}\left[y_i\log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)\right]$$

其中：
- $y_i \in \{0, 1\}$ 是真实标签
- $\hat{y}_i$ 是模型预测的谣言概率

**具体计算**：
```python
# 假设 batch_size=2
logits = [[2.1, -0.5],   # 样本1预测非谣言
          [-1.3, 3.2]]   # 样本2预测谣言
labels = [1, 1]          # 真实：都是谣言

# 样本1：y=1（谣言），但 logits[0][1]=-0.5 很小 → loss 大
# 样本2：y=1（谣言），且 logits[1][1]=3.2 很大 → loss 小
```

#### 5.2 反向传播（梯度下降）

**计算梯度**：
$$\frac{\partial \mathcal{L}}{\partial W} = \frac{\partial \mathcal{L}}{\partial z} \cdot \frac{\partial z}{\partial W}$$

**参数更新**（AdamW）：
$$W_{t+1} = W_t - \eta \cdot \frac{m_t}{\sqrt{v_t} + \epsilon}$$

其中：
- $\eta = 2 \times 10^{-5}$ 是学习率
- $m_t$ 是梯度的一阶矩估计（动量）
- $v_t$ 是梯度的二阶矩估计（自适应学习率）

## 评估指标详细说明

### 1. 准确率（Accuracy）

$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$

**含义**：所有预测中，正确的比例。

**实际结果**：
```python
# 测试集 1996 条数据
TP = 877  # 谣言判对
TN = 943  # 非谣言判对
FP = 53   # 误判为谣言
FN = 123  # 漏判谣言

accuracy = (877 + 943) / 1996 = 0.9118 ≈ 91.18%
```

### 2. 精确率（Precision）

$$\text{Precision} = \frac{TP}{TP + FP}$$

**含义**：在所有**预测为谣言**的样本中，真正是谣言的比例。

**示例**：
```python
precision = 877 / (877 + 53) = 0.943 ≈ 94.3%
```

**物理意义**：
- 高精确率 → 模型很少"误报"
- 适合对误报敏感的场景（如医疗诊断）

### 3. 召回率（Recall）

$$\text{Recall} = \frac{TP}{TP + FN}$$

**含义**：在所有**真实谣言**中，被成功识别出的比例。

**示例**：
```python
recall = 877 / (877 + 123) = 0.877 ≈ 87.7%
```

**物理意义**：
- 高召回率 → 模型很少"漏报"
- 适合对漏报敏感的场景（如反恐检测）

### 4. F1 分数

$$F1 = 2 \cdot \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$

**含义**：精确率和召回率的调和平均，平衡两者。

**示例**：
```python
F1 = 2 * (0.943 * 0.877) / (0.943 + 0.877) = 0.9088 ≈ 90.88%
```

**为什么用调和平均而不是算术平均？**
- 调和平均对极端值更敏感
- 防止一个指标很高但另一个很低的情况

### 5. 混淆矩阵（Confusion Matrix）

```
                预测
              ┌───────┬───────┐
              │  非谣言 │  谣言  │
        ┌─────┼───────┼───────┤
   真   │非谣言│  943  │   53  │ ← FP（假阳性）
   实   │     │  (TN) │  (FP) │
        ├─────┼───────┼───────┤
        │谣言 │  123  │  877  │
        │     │  (FN) │  (TP) │
        └─────┴───────┴───────┘
                ↑
                FN（假阴性）
```

**四个指标的物理意义**：
- **TP (True Positive)**：谣言正确识别为谣言
- **TN (True Negative)**：非谣言正确识别为非谣言
- **FP (False Positive)**：非谣言误判为谣言（**一类错误**）
- **FN (False Negative)**：谣言漏判为非谣言（**二类错误**）

### 6. 实际应用中的指标选择

| 场景 | 优先指标 | 原因 |
|:---|:---|:---|
| 谣言检测 | **Recall** | 漏掉一条谣言可能造成严重后果 |
| 垃圾邮件过滤 | **Precision** | 误拦正常邮件会影响用户体验 |
| 医疗诊断 | **F1** | 误诊和漏诊都要尽量避免 |
